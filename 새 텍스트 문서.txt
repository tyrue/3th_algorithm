스캔용 함수를 따로 만들자
스캔 좌표는 스캔쥐의 현재 위치로 사용하면 될 거 같다.
스캔쥐는 처음에 시계방향으로 오른쪽, 밑, 왼쪽, 위로 간다. 
스캔 한번 뿌리면 스캔쥐가 이동한다. 
만약 스캔을 뿌렸을때 출구를 찾으면 스캔쥐의 위치를 출구 바로 위에 위치 시킨다.

스캔 스택의 필요성? 스택의 의미는 일단 쌓아놨다가 다시 오는건데 굳이 필요할까?
스캔쥐의 스택은 필요함 좌표를 기억해야하기 때문에 또, 출구쪽에서부터 입구쪽으로 와야하기 때문

void scan(int row, int col)
{
	row는 행, col은 열
	이걸 기준으로 3x3위치를 밝힌다.
	출구인지 판단은 이 함수가 판단하자. 뿌리자마자 확인함
	출구의 기준은 벽쪽이고, 1이 아니라 0이어야함
	출구를 찾았다면 이제 출구 확인 변수를 TRUE로 바꿔줌
	이 함수는 그냥 3x3만 보여주면됨
	
}
